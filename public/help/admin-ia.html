Filtre « Masquer les inadaptés »

Un modèle est masqué si l'une de ces deux conditions est vraie :

  1. Il est déprécié — le fournisseur a officiellement retiré ou remplacé ce modèle. Date de dépréciation passée selon le catalogue communautaire LiteLLM.
  2. Il ne supporte pas JSON — le modèle ne sait pas garantir une réponse structurée en JSON. Les workers ont absolument besoin de JSON pour extraire thèmes, mots-clés, etc. Un modèle sans cette capacité est techniquement incompatible.

Ce que le filtre n'utilise pas : le fait qu'un modèle soit activé ou non dans la colonne "Activé". C'est un choix éditorial, pas un critère d'incompatibilité technique.

Dépense estimée

Source des données : la table ai_logs, qui enregistre chaque appel IA réel effectué par l'application (suggestions de mots-clés, thèmes, etc.). Seuls les appels réussis sont comptés.

Formule pour chaque modèle utilisé :

dépense = (tokens envoyés ÷ 1 000 000) × prix entrée
        + (tokens reçus  ÷ 1 000 000) × prix sortie

Les prix (en $/million de tokens) viennent du registre des modèles (ia_model_registry), alimenté par le catalogue communautaire LiteLLM ou par vos retouches manuelles.

Deux limites à connaître :

  • Si un modèle n'a pas de prix renseigné dans le registre, sa dépense n'est pas calculée (affiché —).
  • C'est une estimation basée sur les tokens journalisés côté Lumosphère, pas une facture fournisseur. Les frais réels peuvent légèrement différer selon les arrondis et conditions tarifaires de chaque fournisseur.
